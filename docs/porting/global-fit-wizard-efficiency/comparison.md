# Comparison: selected vs dropped efficiency techniques

This is an internal-refactor study, so "comparison" here contrasts candidate
efficiency techniques (rather than reference programs). Each selected technique
lists mechanism, expected speedup, accuracy risk, and the effort tier(s) it
belongs at. Dropped techniques give a one-line reason.

## Selected

| # | Technique | Mechanism | Speedup | Accuracy risk | Effort tier |
|---|---|---|---|---|---|
| **A** | **Layer-bound pruning** | Fit all-local first; by nested-model monotonicity χ² of any assignment ≥ χ²(all-local). Layer-k IC penalty is known exactly, so stop enumerating layers once `χ²(all-local)+penalty(k) > incumbent+Δ`. | Collapses 2^P → Σ_{j≤~3}C(P,j) (P=7: 128→~30–64). | **None** (admissible; a mis-converged anchor only prunes *less*). | ALL (incl. Exhaustive, with Δ ≥ max robustness delta) |
| **B** | **Cross-template incumbent bound** | Same bound across templates: skip a template whose all-local χ² + min penalty can't beat best IC so far. Order templates by screen score. | Usually only the winning template pays full price. | None (same caveat as A). | ALL |
| **C** | **Strategy-0 / no-Hesse screening fits** | IC needs only χ², not errors. Run interior fits at Minuit strategy 0, loose EDM (~0.1–0.5 in χ²), no Hesse. Hesse + strategy≥1 only on winner + its flip-neighbourhood. | Large constant cut (Hesse is O(k²) fevals; k grows with G). | None to the argmax, *if* step-hint propagation survives Hesse removal (validate). | ALL |
| **D** | **Escalation-on-anomaly (the V killer)** | Interior nodes get one warm-started single-cycle fit. Accept if the monotonicity certificate holds (χ² ≤ parent χ² + ε, valid minimum). Escalate to more variants / cycles / simplex only on failure. Full battery reserved for anchor nodes (all-global, all-local, layer-1). | V 30–80 → ~3–8 on the bulk ⇒ **5–10×**; flattens the V-vs-P growth. | Warm start may sit in a consistent-but-suboptimal basin the multi-start would escape. Exhaustive keeps full multi-start as referee. | Balanced / Thorough (Exhaustive = current) |
| **E** | **Homogeneity (Q) pre-tests** | Per promotable param compute Q = Σ_g(θ_g−θ̄)²/σ_g² ~ χ²_{G−1} from cached single fits. Fix unambiguous tails (huge Q → local, tiny Q → global); enumerate only the ambiguous middle. Band width = the effort knob. | 2^P → 2^(ambiguous) (e.g. 2⁷→2³ = 16×), compounds with A. | Wald errors unreliable for weak/correlated params; compensating variation across a correlated pair. Conservative bands + a flip-recheck on the winner. | Low / Balanced |
| **F** | **Greedy stepwise search** (wire up `_search_parameter_roles`) | From all-global, flip the best single param while IC improves > Δ, then backward-prune. O(P²) fits. | P=7: ~25–30 nodes vs 128; makes P=9–10 templates feasible at all. | Misses interaction pairs (both-local-only wins). Leaves a sparse cache → must fit the winner's flip-neighbourhood for verdicts. | Low (proposal generator at Balanced) |
| **G** | **Wald quadratic surrogate** | From one all-local joint fit + covariance, a closed-form GLS collapse predicts Δχ² of globalising any subset. Score all 2^P in microseconds; real-fit only top-K per layer, expand K if the realised ranking disagrees near the top. | 5–20× fewer real fits at P=7; converts search from fit-bound to verify-bound. | Nonlinearity: the quadratic is wrong when globalising one param moves others far from the all-local solution. Verification step bounds damage; measure surrogate-rank-of-winner. | Balanced / Thorough |
| **H** | **Template racing (successive halving)** | Give all shortlisted templates layers 0–1 only (cheap under A); advance only the top 1–2 to deeper layers. | Only the winner pays full price (compounds with B). | A template losing at layer 0–1 by a wide margin ~never wins deeper. Near-exact. | Balanced / Thorough |
| **I** | **Low-tier portfolio cap** | At Low: shortlist 2–3 templates, hard-cap P≤5, add an explicit complexity prior (extra IC per additive component) so 3–4-component models must earn their place. | Multiplicative on template count and P. | Misses genuine multi-component physics. UI labels it "screening-grade". | Low |
| **J** | **Identifiability demotion** | A cheap conditioning check on the screen fit (parameter correlation ≈1, errors spanning decades) demotes degenerate templates *before* they enter the wavefront. | Attacks the V=77 triexp blow-up at source (that growth is identifiability failure burning restart attempts). | Could demote a marginally-identifiable but correct model; only active sub-Exhaustive. | Low / Balanced |
| **K** | **Screening decimation** | Rebin spectra 2–4× (and/or crop the late-time low-SNR tail) during role search; refit winner + flip-neighbourhood at full resolution for reported verdicts. | Per-fit cost linear in n ⇒ clean 2–4×. | AICc/BIC depend on n → all *compared* scores must share binning. Oscillatory templates need Nyquist-safe binning (gate on the fingerprint FFT). | Low / Balanced |
| **L** | **Profiled / nested locals** (engine) | For fixed globals the joint problem separates into G independent small local-only solves. An outer optimiser over globals only + inner per-dataset solves (parallel over G) replaces one (n_g+n_l·G) Hessian with G small ones. | G-scaling O(G^1.5) → ~linear; the outer problem is tiny. Exact profiling. | Outer objective noisier if inner fits loose. Real engine change — validate vs joint fits. | Engine (every tier benefits) |
| **M** | **Variable projection (VarPro)** | Amplitudes + constant background enter linearly → solve them by linear least-squares inside the objective, removing them from Minuit. Linear *locals* stop multiplying by G. | Removes ~half the nonlinear params in typical models. | Weighting / bounds handling. Accuracy-preserving in principle; k still counts them in the IC. Validate values+errors. | Engine |

## Dropped

| Technique | Why dropped |
|---|---|
| **Branch-and-bound with a per-node IC lower bound** (beyond A/B) | Redundant. The layer bound (A) already gives the only cheap *admissible* bound available: χ²(all-local) is the tightest lower bound computable without fitting. A per-node B&B would need a per-node lower bound that is either the same all-local value (no extra pruning) or requires a fit (defeats the purpose). |
| **`_staged_globalization_assignment` as a search** | Redundant with F (greedy) and E (Q-test). Its cross-group trace-roughness signal is a heuristic cousin of the Q statistic; keep the *idea* as an optional ranker inside E, but do not wire it as a separate search path. |
| **`GlobalSearchOrchestrator` (relaxed→discrete→beam) on the slider** | Not removed from the repo, but **dropped from the delivered slider.** Highest ceiling but zero corpus track record, a second optimiser with its own pathology, and E+F+G+H already cover the Low/Balanced/Thorough spread. Park behind a post-delivery bake-off (see `implementation-options.md` §6); promote to the Balanced engine only if it beats bounded-wavefront on the harness. Wiring it now is speculative scope. |
| **Dataset subsetting at large G** (screen on 5–6 stratified runs) | Low marginal yield once L (profiled locals) makes G ~linear, and it adds a non-monotonic-scan failure mode. The G problem is better solved *exactly* by L than approximately by subsetting. |
| **Simplex-first / gradient-free everywhere** | Simplex is already the rescue path; making it primary is slower per fit for smooth χ². Escalation (D) already invokes it exactly when needed. |
| **Caching across the G-sweep** | Not applicable — node count is flat in G; there is nothing to cache across G that the exact-assignment memo doesn't already hold. |
