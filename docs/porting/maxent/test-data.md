# Maximum Entropy Test Data

## Synthetic Cases For The Core Engine

All synthetic cases generate **raw grouped counts** (not asymmetry):
`N_g(t) = N0_g · exp(-t/τ_μ) · (1 + A_g · s(t - t0)) + bg_g`, Poisson
sampled with a fixed RNG seed, with per-group phases folded into `s`.

1. Single-frequency TF signal, 4 groups at 0/90/180/270°:
   - `s(t) = a · cos(2π f t + φ_g)`, one frequency well inside the window
   - expectation: spectrum peak at `f` (± one bin), fitted phases recover the
     0/90/180/270 pattern up to a global rotation, amplitudes ≈ equal after
     mean-1 renormalization
2. Two close frequencies (separation ~2 FFT bins):
   - expectation: MaxEnt resolves two peaks where the apodized FFT of the
     same data shows one — the motivating use case
3. Field distribution (Gaussian p(B) centered on applied field):
   - expectation: reconstructed lineshape width matches the generating σ_B
     within tolerance; auto window centered on the applied field contains the
     full line
4. Fixed-phase contract:
   - run with phases fixed to the generating values
   - expectation: output phases identical to input (Mantid's `fixPhasesTest`
     contract), amplitudes still refit
5. Background and deadtime nuisances:
   - add constant background to half the groups and a known deadtime τ_d
   - expectation: MODCONST-equivalent recovers the constant offsets;
     deadtime fit recovers τ_d within tolerance when enabled
6. Dead group:
   - one group with all-zero counts
   - expectation: excluded via the bin mask; spectrum unchanged vs the same
     run without the dead group; no NaNs
7. Pulsed-source case (Phase 2):
   - convolve the signal with the parabolic proton-pulse kernel (50 ns
     half-width)
   - expectation: with the pulse response enabled, high-frequency peaks keep
     amplitude; with it disabled, they are attenuated

## Mantid Oracle Data

`scripts/Muon/MaxentTools/` is plain numpy (to be confirmed — open question
2 in `comparison.md`). Strategy:

- run `MULTIMAX` (or the standalone inner `MAXENT`) on the synthetic cases
  above inside a Mantid-capable environment, dump inputs/outputs to `.npz`
  golden files, and commit them under `tests/porting/maxent/` (small arrays
  only)
- record the exact Mantid commit hash and constants (`Tmuon = 2.19704`) in
  each golden file's metadata, since Asymmetry uses CODATA constants and
  per-bin comparisons must compensate or pin constants in the oracle run
- GPL note: golden *data* generated by running Mantid is fine; Mantid code
  must not be committed to this repository

## Reference Datasets

- **MUSR00022725** (ISIS MUSR, used by both Mantid system tests):
  `Testing/SystemTests/tests/framework/MuonMaxEntTest.py` compares against
  reference `MuonMaxEntMUSR00022725.nxs` at tolerance 5e-2, with a 64-row
  phase table. If this run is obtainable (ISIS data archive), it is the
  canonical real-data cross-check.
- **MuonMaxent doc-test values**: `docs/source/algorithms/MuonMaxent-v1.rst:48-64`
  pins five spectrum values (0.110/0.789/0.871/0.821/0.105) for
  MUSR00022725 with Npts=16384, Outer=9, Inner=12, A=0.11, Factor=1.03 after
  `CalMuonDetectorPhases` — a cheap smoke-level cross-check.
- **WiMDA golden files**: run WiMDA manually on a local run (e.g. from
  `~/Source/PSI-data` or an ISIS run already used by the Fourier study),
  save the auto-written `<combiname>.max` spectrum (full parameter header)
  and `<combiname>.mlog` diagnostics log. These lock the WiMDA-parity
  contract: per-cycle χ²/entropy/TEST/sconv rows and final
  phase/amplitude/BG tables.

## Data Contracts To Preserve

- spectrum x-axis canonical unit MHz internally, displayed Field (G) by
  default via `γ_μ/2π = 0.01355342 MHz/G` (matches the Fourier panel)
- counts normalization (1e7-event style) stays internal; outputs are
  normalization-independent
- excluded bins are an explicit mask in the input dataclass, not σ sentinels
- golden comparisons use relative tolerances ≥ the references' own (Mantid
  system tests accept 5e-2) — MaxEnt outputs are iteration-count sensitive,
  so exact equality is only expected against the oracle with identical
  iteration schedules and constants
